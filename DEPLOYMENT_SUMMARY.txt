================================================================================
ROOT CAUSES (23 BUGS FIXED)
================================================================================

1. INCORRECT RESPONSE STRUCTURE ACCESS (8 bugs)
   - Frontend accessed top-level fields instead of nested response object
   - Example: order.symbol → order.response.symbol
   - Affected: 8 data fields across 3 pages

2. INVALID DATE PARSING (3 bugs)
   - new Date(parseInt(order.time)) failed because order.time is undefined
   - Led to "Invalid Date" in activity log and order history
   - Fixed with parseOrderDate() utility function

3. CONFIDENCE SCORE TYPE MISMATCH (5 bugs)
   - Accessed confData.confidence_score (doesn't exist)
   - Should use confData.total_score (already 0-100)
   - Was multiplying by 100 again → NaN or 8500%

4. CONFIDENCE METRICS PATH ERROR (3 bugs)
   - Accessed confData.metrics.volatility_index (wrong path)
   - Should use confData.breakdown.volatility
   - All 3 metrics (volatility, depth, proximity) had wrong paths

5. GRADE RECALCULATION (1 bug)
   - Recalculated grade ignoring backend-provided grade
   - Now uses backend grade directly

6. WIN RATE CALCULATION (1 bug)
   - Accessed o.realizedPnl which doesn't exist
   - Now calculates from available data

7. LAST ORDER STATUS ACCESS (1 bug)
   - Accessed wrong nested level for status
   - Now uses extractOrderDetails(lastOrder).status

8. MISSING NULL SAFETY (Multiple)
   - Added optional chaining (?.) and nullish coalescing (??)
   - Prevents "undefined" values from appearing
